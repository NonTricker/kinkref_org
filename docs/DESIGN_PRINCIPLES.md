# 設計原則 — kinkref.org

> **版本**：v1.0（公開版）
> **最後更新**：2026-05-02
> **License**：CC-BY-SA 4.0

本文件說明 kinkref.org 的網站設計原則。給設計師、前端工程師、內容策展人、未來接手者參考。

> 本文件**不涉及視覺設計**（色彩、字體、版式由獨立的視覺指南另行制定）。本文件聚焦於**資訊架構、互動規範、內容邊界**。

---

## 一、整體設計基調

**這是一個學術參考工具。不是社群平台、不是部落格、不是情色站。**

### 1.1 視覺語言對標

- ✅ 對標：MDN Web Docs、Arch Wiki、Stanford Encyclopedia of Philosophy
- ❌ 不對標：任何運動風、社群風、霓虹／皮革／繩索意象的站

### 1.2 核心原則

| 原則 | 內涵 |
|------|------|
| **匿名可查閱** | 沒有帳號、沒有登入、沒有「加入」任何東西。這是刻意的設計選擇，不是技術限制 |
| **低門檻進入** | 使用者不需要表態、不需要承認任何身分，就能取得資源 |
| **高密度資源** | 內容本身嚴謹、有來源、有結構，不是無深度的 listicle |
| **Privacy-respecting** | 無追蹤、無 cookie 個人化、無 analytics（或僅 Plausible 等隱私尊重方案） |
| **語氣中性** | 描述性，不評價哪種傾向「正確」或「進階」 |

### 1.3 字體與色彩方向

由視覺設計階段決定，但須符合：
- 內文優先**可讀性**（襯線或無襯線皆可）
- 中英混排要順
- 中性、乾淨、高對比、學術感但不無聊
- 深色／淺色模式皆需支援

## 二、頁面架構

### 2.1 首頁（Landing）

**做**：
- Tagline 中英並列（英文在上、中文在下）
- 三柱入口卡片：術語表 / 文獻指南 / 書架清單
- 三類讀者導引：「我是入門者」「我要引用」「我在做研究」
- 最近更新：3-5 筆內容更新（帶日期）
- Footer：License、GitHub repo、聯絡 email

**不做**：
- ❌ 站主照片 / 頭像 / 個人 logo
- ❌ Hero image（無 banner、無意象圖）
- ❌ 社群媒體連結
- ❌ 捐款按鈕 / 金流入口
- ❌ Newsletter 訂閱
- ❌ 跑馬燈 / 公告欄

---

### 2.2 術語表（Glossary）

**做**：
- 字母 / 注音排序的索引頁
- 每筆術語條目：
  - 中文譯名（主標）+ 英文原文（副標）
  - 一句話定義（≤30 字，給快速查閱者用）
  - 完整定義（150-400 字白話解釋）
  - 來源年代 + 首見文獻 + DOI
  - 相關術語交叉連結（如 Master ↔ Owner ↔ Dominant）
  - 與相近術語的辨析（disambiguation）
  - 外部標準對應（如 bdsmtest.org archetype）
- 全站搜尋（至少 glossary 內可搜）
- 每個術語頁有固定 URL（可被外部引用）

**不做**：
- ❌ 留言 / 評論區
- ❌ 「你覺得這個定義準確嗎」投票
- ❌ 個人經驗分享欄位
- ❌ 圖片插圖（術語頁純文字 + 連結）

完整 schema 見 [`CONTENT_SCHEMA.md` § 5](./CONTENT_SCHEMA.md)。

---

### 2.3 文獻指南（Annotated Bibliography）

**做**：
- 按主題分類：consent / psychopathology debate / aftercare / cross-cultural / harm reduction / methodology / legal / neuroscience / ...
- 每篇文獻條目：
  - 論文標題（英文原文）+ 作者 + 年份
  - 中文摘要（150-400 字，策展人撰寫）
  - 策展人導讀（1-5 句）
  - DOI 連結 + Open Access 標記
  - 取得指引（paywalled 時提示 preprint 位置）
  - 主題標籤 + 讀者標籤（可交叉篩選）
  - 翻譯狀態：「有全文翻譯」/「僅摘要」/「計畫翻譯中」
- APA 格式一鍵複製
- Zotero / BibTeX export（如技術可行）
- Reading Roadmap 頁面：為三類讀者各列建議閱讀順序

**不做**：
- ❌ 嵌入論文全文 PDF（版權問題）
- ❌ 讀者評分 / 排名
- ❌ 「推薦類似論文」演算法推薦
- ❌ 留言討論區

完整 schema 見 [`CONTENT_SCHEMA.md` § 7](./CONTENT_SCHEMA.md)。

---

### 2.4 書架清單（Curated Reading List）

**做**：
- 按 `book_type` 分類：academic / popular / fiction / memoir / reference
- 每本書條目：
  - 書名（原文 + 中譯如有）+ 作者 + 出版年
  - 類型標籤 + 主題標籤
  - 中文評介（100-300 字）+ 策展人推薦語（1-5 句）
  - 取得方式（博客來 / Amazon / 圖書館 / 電子書平台連結）
  - 語言（原文語言 + 是否有中譯本）
- 可按分類、主題、語言篩選

**不做**：
- ❌ 讀者書評 / 心得投稿
- ❌ Affiliate 導購（不賺佣金）
- ❌ 書封圖片（版權問題，除非取得授權）
- ❌ 評分星等

完整 schema 見 [`CONTENT_SCHEMA.md` § 6](./CONTENT_SCHEMA.md)。

---

### 2.5 全文翻譯頁（選擇性區塊）

**做**：
- 僅限已取得授權（Open Access CC-BY 或作者 preprint 授權）的論文
- 翻譯頁面頂部必有：
  - 原文引用資訊（完整 APA 格式）
  - DOI 連結
  - 授權說明（翻譯依據版本 + 授權類型）
  - 翻譯版本號 + 最後審校日期
  - 「AI-assisted, human-reviewed by [署名]」標示
- 中英對照模式（可選：並列或切換）
- 方法論段落特別標註（提醒讀者此段經人工複審）
- 讀者前置說明（策展人導讀 + 建議前置閱讀）

**不做**：
- ❌ 翻 paywalled 論文全文
- ❌ 機器翻譯直出（必須有人工審校標記）
- ❌ 讀者直接編輯翻譯（改走 GitHub PR）

### 2.6 Directory（資源指路）

> **設計重點**：本站不對任何個人或組織進行推薦或背書。Directory 是中性資訊聚合，不是品質保證。

**做**：

#### 大學社團
- 表格形式
- 欄位：社團名稱 / 所屬學校 / 官方聯絡方式 / 是否公開招生 / 最後確認日期
- **不寫**：活動內容、成員、評價

#### Kink-friendly 專業人士
- 僅限「**本人主動申請**列入」者
- 欄位：姓名或代稱 / 專業（諮商師 / 律師 / 醫師）/ 聯絡方式 / 本人同意標記
- **不做**：推薦語、評價、認證

#### 活動行事曆 aggregator
- 僅列外部來源連結
- 不自建活動資料庫
- 頁面頂部固定免責聲明：
  > 「本站不主辦、不審核、不背書任何活動」

**不做**：
- ❌ 場地 / 工作室
- ❌ 個人 Dom / Sub / 繩師
- ❌ 地圖標記
- ❌ 活動報名 / 售票
- ❌ 「推薦」或「認證」標章

---

### 2.7 About 頁

**做**（兩段話以內）：
- 第一段：站的定位（一句話）
- 第二段：策展人身分（圈名 + 一句自我定位）
- License 說明（CC-BY-SA 4.0）
- 聯絡方式（獨立 email alias）
- GitHub repo 連結
- 「如何貢獻」連結

**不做**：
- ❌ 策展人照片 / 頭像
- ❌ Founder story / Manifesto
- ❌ 個人社群連結
- ❌ Team 頁面

## 三、全站共通元件

### 3.1 必須具備

- **全站搜尋**（至少覆蓋 glossary + bibliography + reading list）
- **麵包屑導航**（breadcrumb）
- **深色 / 淺色模式切換**
- **響應式設計**（mobile-first）
- **每頁底部**：
  - 「在 GitHub 上編輯此頁」連結
  - 最後更新日期
  - License 標示
- **`llms.txt`**（站根目錄，對 AI crawler 開放）
- **`robots.txt`**（允許搜尋引擎索引）
- **SEO meta**：每個術語頁、文獻頁都要有獨立的 title / description / OG card
- **年齡確認閘門（Age Gate）**：首次訪問時彈出，確認 18 歲以上。樣式簡潔不色情，語氣中性（如「本站內容涉及成人議題的學術討論」）

### 3.2 嚴格禁止

- ❌ 使用者帳號系統
- ❌ 登入 / 註冊
- ❌ Cookie-based 個人化
- ❌ 商業 analytics 追蹤（或僅用 Plausible 等隱私尊重方案）
- ❌ 彈窗廣告
- ❌ AI chatbot
- ❌ 任何色情意象（站內零圖片或僅用中性圖表）

## 四、互動機制

### 4.1 互動三類型

| 類型 | 說明 | 主要管道 | 備用管道 |
|------|------|---------|---------|
| **A. 內容評論** | 對特定書目／論文的讀後感、補充、異議 | 投書 → 讀者來信 | — |
| **B. 內容建議** | 建議新增某篇論文 / 書目 | GitHub PR | 投書 |
| **C. 自薦入站** | 大學社團、kink-friendly 專業人士申請列入 Directory | 自薦表單 | — |

---

### 4.2 內容評論：投書 → 讀者來信

**設計邏輯**：不是開放留言區，是「雜誌讀者來信」格式。投書由策展人審核後，以 named annotation 形式發布在對應頁面底部。

#### 投書表單欄位

| 欄位 | 必填 | 說明 |
|------|------|------|
| 類型 | ✅ | 「對此書目／論文的評論」 |
| 針對的內容 | ✅ | 搜尋／選取對應頁面 |
| 留言內容 | ✅ | 上限 500 字 |
| 你的稱呼 | ✅ | 圈名、真名、或「匿名讀者」 |
| SNS／聯絡方式 | ❌ | 選填。填寫視為同意一併顯示 |
| 同意聲明 | ✅ | 勾選同意可能完整發布 |

#### 審核流程

```
投書送出
  → 站主信箱收到（不公開）
  → 站主每月定期 review
  → 通過：在對應頁面底部「讀者來信」區域完整發布
  → 不通過：email 一句拒絕原因
```

#### 通過標準

- ✅ 針對內容本身的補充、異議、延伸閱讀建議
- ✅ 跨文化視角
- ✅ 術語使用上的勘誤
- ❌ 針對個人的評論
- ❌ 個人經驗敘事
- ❌ 廣告 / 宣傳
- ❌ 涉及未成年的任何描述

#### 顯示格式

```
── 讀者來信 ──────────────────────
[稱呼]（@SNS，如有提供）
「⋯⋯留言內容⋯⋯」
發布日期：YYYY-MM    審核人：[圈名]
─────────────────────────────────
```

每頁底部最多顯示 3 則，超過則折疊。

---

### 4.3 內容建議：PR 優先，投書備用

- **路線 1（有程式基礎）**：GitHub PR
- **路線 2（無程式基礎）**：投書，類型選「建議新增內容」

---

### 4.4 自薦入站：Directory 自薦表單

#### 自薦表單欄位

| 欄位 | 必填 | 說明 |
|------|------|------|
| 類型 | ✅ | 大學社團 / 諮商師 / 律師 / 醫師 / 其他 |
| 名稱或代稱 | ✅ | 社團名 / 個人執業名 / 圈名均可 |
| 官方聯絡方式 | ✅ | 用於後續確認 |
| 公開聯絡方式 | ✅ | 列在 Directory 上的那個 |
| 官方證明連結 | ❌ | 學校網頁 / 執照查詢 |
| 自我說明 | ✅ | 上限 200 字 |
| 同意聲明 | ✅ | 勾選自願申請列入 |

#### 審核流程

```
自薦表單送出
  → 站主信箱收到
  → email 做基本確認（對方真實存在）
  → 通過：列入 Directory，標記「最後確認：YYYY-MM」
  → 每 12 個月重新確認
  → 未回覆 → 標記「待確認」→ 30 天未回 → 移除
```

---

### 4.5 互動禁區

- ❌ 即時留言（不是論壇）
- ❌ 讀者投票 / 按讚
- ❌ 公開的審核狀態追蹤
- ❌ 自動發布（所有互動內容必須人工審核後上線）

## 五、內容禁區（永遠不出現）

設計時不需要為以下內容預留版面：

- 場地 / 工作室介紹
- 個人背書或推薦
- 安全指引 / 操作教學
- 未經審核的使用者自由留言（有策展的讀者來信 ≠ 自由 UGC）
- 金流 / 捐款 / 商品
- 即時社群討論區
- 即時通訊 / 聊天室
- 活動報名
- 配對 / 交友功能

## 六、未來考慮的補充功能（Parking Lot）

以下是規劃中、尚未實作的功能，列在這裡供未來參考：

### 6.1 FAQ 頁面

「使用場景導向」的 FAQ，例如：
- 「我剛做完 bdsmtest.org，結果說我有 X 傾向，這代表什麼？」
- 「我的伴侶跟我說想嘗試 BDSM，我從哪裡開始了解？」
- 「我在寫關於 BDSM 的論文，有沒有推薦的入門文獻？」
- 「我是諮商師 / 醫師，遇到了圈內個案，有哪些參考資源？」

### 6.2 「給親密他人」的 pathway

針對發現家人 / 伴侶接觸 BDSM 但自己不了解的人，提供專屬入口：
- 去病理化文獻
- BDSM ≠ 虐待的核心判準
- kink-friendly 諮商師 Directory 連結

### 6.3 術語頁的「詞彙地圖」

每個術語頁底部加一個靜態詞彙家族圖，顯示譜系：

```
Dominant
  ├── Master（long-term D/s, O&P 文化）
  ├── Sir（leather 傳統）
  ├── Daddy Dom（CG/L 脈絡）
  └── Owner（財產動態）
```

### 6.4 翻譯頁的讀者導引

每篇翻譯頂部加策展人的 1-2 句導引：「這篇寫給：⋯⋯」「建議先讀：⋯⋯」「方法論段落較技術性，可跳過」。
